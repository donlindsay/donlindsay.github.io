Stress + telemetry kit

Tools for stress-testing the local-LLM stack and capturing GPU/system telemetry.

Scripts

  • restart-cold.sh — clean ordered restart of the whole stack to a cold baseline

(reload AllegroGraph, clear Ollama VRAM, reset vLLM, app services up, vLLM left down).

  • capture.sh — background telemetry sampler:
  • capture.sh start <label> [interval_s] → begins GPU (nvidia-smi) + system (load/RAM) CSVs under runs/
  • capture.sh stop · capture.sh status · capture.sh list
  • stress.sh — load generator via the ai-api (:8765):
  • stress.sh bench <engine> <model> [n] — sequential benches, avg tok/s
  • stress.sh load <conc> <count> [eng] [mdl] — N requests at given concurrency → req/s
  • stress.sh soak <seconds> <conc> [eng] [mdl] — keep workers busy for a duration
  • stress.sh matrix [--vllm] — one bench per non-placeholder engine×model cell
  • stress.sh vllm on|off — load / release the vLLM service

Typical run


cd /opt/aistack/telemetry
./restart-cold.sh                 # optional: reset to cold baseline
./capture.sh start run1 1         # begin telemetry @1s
./stress.sh soak 60 4             # 60s, 4 concurrent workers on ollama×nemotron
./capture.sh stop                 # writes runs/<ts>_run1.gpu.csv + .sys.csv

To stress vLLM: ./stress.sh vllm on (waits for load), then ./stress.sh soak 60 8 vllm twin-llama, then ./stress.sh vllm off.

CSV columns

  • gpu: timestamp, mem.used MiB, mem.free MiB, util.gpu %, util.mem %, temp °C, power W, sm clock MHz, mem clock MHz
  • sys: epoch, load1, mem_used_mib, mem_avail_mib

Engine × model cells (free matrix)

Active matrix (2026-06-08): {ollama, llamacpp} × {nemotron, twin-llama} — llamacpp×nemotron (default, live) · llamacpp×twin-llama (live once swapped) · ollama×nemotron (live) · ollama×twin-llama (live, llama3.2 pulled). vllm× is shelved (the stress.sh vllm on|off / matrix --vllm helpers still drive it on demand for benchmarking). tabby× is a placeholder skin.
