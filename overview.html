AI Stack — Overview

id: urn:kg:doc:overview · source: guide · date: 2026-06-08

The AI Stack is a local-first, offline-capable AI workbench: a knowledge graph and a language model running entirely on one Windows + WSL2 machine, driven by a guitar-pedalboard desktop UI. No cloud is required for the core loop — chat, retrieval, and the knowledge graph all run on-box.

At a glance

  • One runtime — llama.cpp. llama-server (built for CUDA sm_120 / RTX 5070 Blackwell) serves

both chat (Nemotron-3-Nano 4B, with Twin-Llama as a second model) and, through an in-process embedding adapter, the vectors behind retrieval. Ollama, vLLM, n8n and ComfyUI are optional, shelved add-ins.

  • AllegroGraph 9.0.1 is the knowledge graph and RAG store — documents, agent memory, and

provenance live as RDF, with native vector search over them.

  • Pedalboard UI. A native Avalonia app (skin: Easy) presents the stack as effect pedals —

Chat, Ask, Search, Graph, Image, KG, VXA (voice/audio), and SHELL (an agentic coding console).

  • Provenance everywhere. Agent actions and shell operations are logged to a chain-of-custody

graph (PROV) — the stack can explain what it did and why.

What it does

  • Chat with a local model, with optional long-term memory.
  • RAG over your documents (data.docs) with cited answers.
  • Knowledge graph queries (SPARQL / natural-language-to-SPARQL) over a federated AllegroGraph.
  • Image generation via a Stable Diffusion worker.
  • Agentic shell — the SHELL pedal runs real commands and edits files under a permission gate,

and can even modify the stack's own source (self-building) with a human approving every change.

See the Guide to run it and the Architecture for how it fits together.
