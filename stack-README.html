Local AI Stack — native (WSL2 Ubuntu, no Docker)

Everything runs as native host processes. Start it all (idempotent):


bash /opt/aistack/start-stack.sh

Nothing auto-starts on boot — run the script after a reboot.

The brain: one model — nemotron-3-nano:4b (NVIDIA, 2.8 GB, Blackwell GPU) — runs the agent, generation/entity-extraction, and AllegroGraph's documents + tables; nomic-embed-text does embeddings. That is the entire model roster.

Layout

  • agraph/ — AllegroGraph install + 19.9 GB data (lifted from the old container). Config agraph/etc/agraph.cfg.
  • n8n-data/.n8n/ — n8n user folder: database.sqlite (workflows+credentials), encryption key in config.
  • n8n-runner/ — extracted task runner: task-runner-python/ (run from build/lib), venv/ (py3.12 + websockets/urllib3/ddgs/paramiko/agraph-python), run-python-runner.sh (auth-handshake wrapper that replaces the launcher), task-runner-javascript/ (kept for future JS Code nodes).
  • graphrag/ — git repo: pipeline, n8n workflow builders (n8n/build_*_wf.py), extract/ (web RDFizer rack), shapes.ttl, pre-commit.
  • ComfyUI/, models/ (ollama + huggingface + tabby caches).
  • main/ — the unified console app (own git repo): the aistack backend package +

desktop/web/cli frontends. Web served at :7860 as ai-console; desktop + cli on demand.

  • legacy/ — the retired standalone console/ (web) and desktop/ (Qt) apps, superseded by main/.
  • legacy-docker/ — retired Docker-era scripts (reference only; not used).
  • n8n-export/ — workflow + credential backup taken during the migration.
  • Helper scripts: start-stack.sh, run_shacl.sh, gruff_friendly.py, rebase_urn.py, rdf_remedies_migrate.py, n8n_rag_index.def.

n8n webhooks

POST /webhook/{ollama-gen, agraph-rag, agraph-rag-native, comfy-image, websearch, extract, claude, perplexity} and hosted chat UIs /webhook/{chat-ollama, chat-rag, chat-image, chat-search}/chat. claude (Anthropic) and perplexity are cloud APIs — those calls leave the machine.

Web extraction (Sponger-style RDFizer)

graphrag/extract/ is a native take on Virtuoso's Sponger cartridges: it turns a list of URLs into RDF in AllegroGraph via three best-effort tiers — extruct (JSON-LD / microdata / RDFa / OpenGraph structured data), trafilatura (main content + metadata), and nemotron (salient entities). Each page becomes a RAG doc (so /webhook/agraph-rag retrieves it) plus KG metadata + entity nodes on urn: IRIs. Add bespoke per-domain extractors via the CARTRIDGES registry in extract/rack.py.


curl -X POST http://localhost:5678/webhook/extract -H 'content-type: application/json' \
  -d '{"urls":["https://example.com","docs.vllm.ai"],"repo":"n8n_rag","limit":10,"llm":true}'
