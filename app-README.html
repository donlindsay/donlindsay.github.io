AI Stack — native front end (Avalonia starter)

A minimal Avalonia (C#/.NET 8) desktop client for the local AI Stack. It talks to the Python backend purely over HTTP — the FastAPI shim at http://127.0.0.1:8765 (/opt/aistack/main/api.py) — so the UI and the backend stay completely decoupled. WSL2 forwards localhost, so this Windows app reaches the in-WSL API with no extra setup.

This is a starter, deliberately small: a dark-skinned chat window that

  • streams your question to the agent (POST /chat),
  • renders the reply, and shows chart / graph / generated images inline (decoded from

the data-URI the API returns),

  • shows live service status in the header (GET /status).

It proves the architecture end to end. The other capabilities (RAG, web search, extract, SPARQL, NL→SPARQL) are already exposed by the shim — add a tab per endpoint the same way.

Prerequisites

  • .NET 8 SDK — https://dotnet.microsoft.com/download (VS 2022 installs it for you)
  • The stack running in WSL with the API up:

```bash

# in WSL (Ubuntu, as root) /opt/aistack/main/run.sh web # or the full ./start-stack.sh — both bring up ai-api

# or just the API alone: cd /opt/aistack/main && ./venv/bin/python main.py api

`` Confirm from Windows: curl http://127.0.0.1:8765/status`

Run it

VS Code (with the C# Dev Kit + Avalonia for VS Code extensions): open this folder, then dotnet run in the terminal — or F5 to debug. The Avalonia extension gives you a live XAML previewer.

Visual Studio 2022 (install the Avalonia for Visual Studio extension for the designer): open AiStackNative.csproj and press F5.

Command line:


cd C:\Users\donal\ai-stack-native
dotnet run

Why Avalonia

Chosen because it's first-class in both Visual Studio and VS Code, its CSS-like styling suits a custom "guitar-pedal" skin, and it can host a WebView later to embed Gruff live. Swapping to WPF is a drop-in on the same shim (Windows-only, best VS designer, native WebView2) — only this C# project changes; the Python backend does not.

Embedded Gruff — the GruffView companion (WebView2)

Gruff is embedded in a native window — just not in the Avalonia process. Embedding a WebView2 control directly in Avalonia (WebView.Avalonia) segfaults on instantiation (0xC0000005) on both Avalonia 11.2.1 and a version-matched 11.0.11, with the WebView2 runtime installed and WebView2Loader.dll deployed — i.e. the package's native interop is broken on .NET 9, not a runtime/version problem. So Gruff runs in GruffView/, a tiny WPF app using the official, rock-solid Microsoft.Web.WebView2.Wpf control. The Avalonia KG tab's Open Gruff (window) button launches GruffView.exe (passing the AG URL); it loads the live AllegroGraph WebView/Gruff embedded. This is the WPF-WebView2 strength we picked Avalonia despite — proof that the shim cleanly supports a second native shell where it's the better tool.


ai-stack-native/
  AiStackNative.csproj   # Avalonia app (6 tabs, pedal skin)
  GruffView/             # WPF companion: WebView2 → Gruff  (net9.0-windows)

GruffView is excluded from the Avalonia project's globs (see the <Compile Remove> in AiStackNative.csproj); build it separately with dotnet build GruffView.
