vLLM serving — ⚠️ SHELVED (dormant, 2026-06-08)

SHELVED. vLLM is no longer in the active Slot-0 engine matrix or the default — llama.cpp (deploy/llamacpp/) is the primary GPU engine now. This serving infra is preserved and still swap-in-able via set_engine("vllm"); the ai-vllm service is left stopped to free the GPU. Re-promote by re-adding "vllm" to localllm._ENGINES. The notes below are historical (last role: the vLLM × {Nemotron, Twin-Llama} quadrants).

(historical) vLLM serving — "Twin Llama" Chat pedal (swap-in for Ollama/Nemotron)

OpenAI-compatible server on 127.0.0.1:8000, served-model-name twin-llama.

  • serve.sh — runtime launcher (lives at /opt/aistack/vllm-worker/serve.sh).

Loads unsloth/Llama-3.2-3B-Instruct (ungated mirror of Meta Llama 3.2-3B; the gated meta-llama repo needs a license click the HF token lacks). Swap weights via VLLM_REPO=... (e.g. unsloth/Llama-3.1-8B-Instruct).

  • ai-vllm.service — systemd unit (installed to /etc/systemd/system).

Not enabled at boot (VRAM-heavy). On-demand: systemctl start ai-vllm.

Blackwell sm_120 (RTX 5070) notes

vLLM 0.22.1 + torch 2.11.0+cu130. flashinfer's cu13 cccl headers fail to JIT-compile on sm_120, so it is bypassed:

  • VLLM_USE_FLASHINFER_SAMPLER=0 -> torch-native sampler
  • attention auto-selects FLASH_ATTN (prebuilt, no compile)

GPU budget --gpu-memory-utilization 0.70 (~10.6GB used, ~1.3GB free) so the WSLg compositor keeps headroom. These are swap engines: Twin Llama owns the Chat slot when active; Nemotron (Ollama) idles/unloads.
