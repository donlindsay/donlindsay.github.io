DEVLOG SNAPSHOT — AI Stack — 2026-06-07

A private, offline-first knowledge-and-tools workbench running natively in WSL2 Ubuntu. Two pillars: AllegroGraph (symbolic brain, CPU/RAM) + Ollama (neural shell, GPU/VRAM), with a diffusers Stable Diffusion worker and a FastAPI/Gradio + Avalonia frontend.

1. Last 24 hours

  • Pedalboard: rebranded the image pedal ComfyUI → Stable Diffusion; built a generic swappable-pedal mechanism (double-click → SWAP PEDAL palette). Pairs: SD ↔ ComfyUI (image), SerpAPI ↔ n8n (search).
  • gowf wildland-fire KG: cloned, loaded 12 Turtle files into data.fire (named graphs per file), federated under fed.kb.
  • KnowWhereGraph: imported the modular KWG ontology into schema.kwg (572 classes); authored a gowf↔KWG alignment; wired live SPARQL SERVICE federation.
  • Remediation R1–R7: governance set — KWG canonical, gowf sandbox. Reversed the alignment to gowf ⊑ KWG, deprecated duplicates, adopted KWG-SHACL as the conformance gate (measured the sandbox as structurally disjoint).
  • Cloud council: grew the cloud web tier from 5 → 8 peers — serpapi, ddgs, perplexity, claude, openai, then +openrouter (gemini-2.5-flash), +huggingface (llama-3.3-70b), +ollama (gpt-oss:120b via the linked box). Fan-out in parallel, synthesized locally by nemotron.
  • Keys: provisioned SerpAPI, OpenRouter, Hugging Face, Ollama Cloud; linked the box to ollama.com via its device pubkey; closed the Perplexity rotation end-to-end; retired/scrubbed gitinfo.txt.
  • Credential inventory: materialized the vendor → access → service → model dependency graph into the registry repo with custody events.
  • n8n eviction: replaced the 3 n8n-routed paths (Claude, Perplexity, web search) with direct vendor calls; then folded n8n into aistack/n8n.py and made the Search pedal a SerpAPI ↔ n8n swap.
  • Purge & disk: deleted n8n-data, the Python runner, and nvm Node 22 (~3.2 GB); reclaimed ~22 GB total (apt cache, unused llama3.1:8b model, dead Tabby cache, log bloat, legacy frontend repos); added an rsyslog filter to silence WSLg syslog spam.

2. Full software audit

  • OS / kernel: Ubuntu on WSL2, kernel 6.6.87.2-microsoft-standard-WSL2. No Docker in the runtime (installed, 0 containers).
  • Symbolic tier: AllegroGraph 9.0.1 (free, 5M-triple license) embedding Allegro CL; SPARQL + RDFS++ reasoning + Prolog (custody lineage) + SHACL + native vector store.
  • Neural tier: Ollama 0.30.0 — local models nemotron-3-nano:4b (agent + council synthesis), nomic-embed-text (embeddings); + Ollama Cloud (gpt-oss:120b) via the linked device key.
  • Image tier: diffusers Stable Diffusion worker — torch 2.11.0+cu128 (sm_120 kernels), diffusers 0.38, transformers 4.57.6; socket-activated + idle-exit to free VRAM; multi-checkpoint.
  • Backend: Python 3.12.3 — FastAPI 0.136.3, uvicorn 0.49.0, Gradio 6.16.0, requests 2.33.1, pydantic 2.13.4, numpy 2.4.6, ddgs 9.14.4, matplotlib, pillow. Package aistack (core, skills, council, federation, rag, memory, custody, registry, jsonld, sdgen, n8n).
  • Frontend: Avalonia / .NET 9 desktop "pedalboard" (Windows) + the Gradio web console (:7860) + the FastAPI native API (:8765).
  • Tooling: GitHub CLI 2.93.0, sqlite3 3.45.1, node v18.19.1 (distro; pandoc dependency only), rsyslog 8.2312.0.
  • Evicted / removed: n8n + its nvm Node 22, ComfyUI, Tabby (code-LLM cache), Docker (unused).

5. Novel UI developments

  • Guitar-pedalboard metaphor — each capability is an effect pedal: Chat=Overdrive, Ask=Reverb, Search=SerpAPI, Graph=EQ, Image=Stable Diffusion, KG=Chorus, VXA=Volume. Stomp to engage (selects its page); LED shows that service's health.
  • Swappable pedals — double-click any slot with alternates to pop a "SWAP PEDAL" palette and re-mount a different pedal in place (generic PedalVariants mechanism). Live pairs: Image SD↔ComfyUI, Search SerpAPI↔n8n; legacy skins honestly show an OFF LED.
  • Per-pedal knobs — a MAIN/OPTS/SYSTEM mode knob + a live parameter knob with a value readout (steps, docs, hits, wpm).
  • ACCESS knobs — WEB (OFF/LOCAL/CLOUD, where CLOUD convenes the 8-peer council) and SHELL (NONE/UBUNTU/WIN).
  • Cloud council as UI — engaging WEB→CLOUD fans a query to 8 peers, synthesized locally.
  • Brushed-metal / walnut textures, glossy enclosures, theme skins, a scripted demo auto-tour, and the VXA voice pedal.

6. Full hardware audit

  • CPU: AMD Ryzen 9 9900X (Zen 5), 12 cores presented to WSL; AVX-512 (incl. bf16, VNNI, VP2INTERSECT), SHA-NI, AES.
  • Memory: 23 GiB RAM in WSL (capped from a 31 GB host to protect AllegroGraph), 8 GiB swap. Headroom: ~16 GiB available at audit.
  • GPU: NVIDIA GeForce RTX 5070 — 12 GB GDDR7, Blackwell sm_120 (compute capability 12.0), driver 595.79, CUDA 13.2. Idle ~4% at audit.
  • Disk: 1 TB (1007 G) — 59 G used, 898 G free (7%).
  • Silicon mapping: symbolic (AllegroGraph, Whisper int8) on CPU/RAM; neural (Ollama LLM, SD worker) on GPU/VRAM. VRAM is freed on SD idle-exit and Ollama idle-unload so the 12 GB is shared, not contended.
