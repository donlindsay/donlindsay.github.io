DEVLOG SNAPSHOT — AI Stack — 2026-06-08

A private, offline-first knowledge-and-tools workbench running natively in WSL2 Ubuntu. Two pillars: AllegroGraph (symbolic brain, CPU/RAM) + local LLM engines (neural shell, GPU/VRAM), with a diffusers Stable Diffusion worker and a FastAPI/Gradio + Avalonia frontend. This snapshot covers the local-LLM expansion: a second serving engine, a two-axis pedalboard, and the Reaper VRAM/performance console.

1. This session

  • vLLM "Twin Llama" serving — LIVE on Blackwell sm_120 (RTX 5070). Stood up a second local

serving engine beside Ollama. The hard part was sm_120 (compute 12.0): pinned an isolated venv to vllm 0.22.1 + torch 2.11.0+cu130 + torchvision 0.26.0+cu130, put the bundled cu13 nvidia libs on LD_LIBRARY_PATH, and — crucially — bypassed flashinfer (its cu13 cccl headers won't JIT-compile on Blackwell) with VLLM_USE_FLASHINFER_SAMPLER=0, after which vLLM auto-selects the prebuilt FLASH_ATTN backend. Served the ungated unsloth/Llama-3.2-3B-Instruct mirror under a stable twin-llama alias (the gated meta-llama repo wants a license click). --gpu-memory-utilization 0.70 keeps ~1.3GB WSLg headroom. systemd unit ai-vllm (on-demand, not enabled at boot).

  • On-demand vLLM lifecycle. localllm.set_engine() best-effort systemctl start ai-vllm on swap

to vLLM, stop on swap away — only one local engine resident at a time. Verified: vLLM→Ollama releases the GPU (946 MiB used / ~11 GB free).

  • 6-slot pedalboard restructure. The board went from 7 flat pedals to a 6-slot board (0 top → 5

bottom): 0 Engine (Ollama/vLLM/Tabby), 1 Model (Nemotron/Twin Llama), 2 Knowledge (one synthesized pedal whose carousel faces are Ask/Search/Graph/KG), 3 Media (Stable Diffusion/ComfyUI), 4 Reaper, 5 Audio/VXA. Architecturally, pedal-slot is now decoupled from content-tab (PedalSpec.Tab, _activeSlot, ContentTab): Engine+Model both drive the Chat pane; Knowledge's active face selects its pane; the 7 original panes are reused, with one new Reaper pane added. Engine and Model are independent axes — a free engine × model matrix.

  • Engine × model free matrix (backend). localllm carries two axes; _resolve(engine,model) is

the single source of truth shared by generate() and bench(). Live pairs: Ollama×Nemotron and vLLM×Twin Llama; cross-cells fall back gracefully to Ollama (Ollama currently holds only nemotron-3-nano:4b, so Ollama×Twin Llama falls back until a llama is pulled). POST /local-engine now takes optional engine and/or model.

  • Reaper — VRAM manager + performance console (slot 4). Upgraded from a status readout to a

full dashboard: live GPU telemetry (VRAM bar, util %, temp, power via nvidia-smi), a clickable 3×2 engine × model status matrix (live / loadable / fallback / placeholder, each showing its resolution), on-demand Load/Release vLLM, and per-cell benchmarks. Benchmarks use Ollama's native nanosecond timing counters for exact metrics — measured 132.8 tok/s · TTFT 26ms · load 159ms for nemotron; the vLLM path uses wall-clock + completion tokens. New endpoints: GET /gpu, GET /local-matrix, POST /local-bench.

  • Housekeeping. Closed the GitHub n8n-credential PAT rotation (user revoked provider-side; the

old token in the n8n-export is inert). Recorded a standing demo-capture choreography directive (natural scroll/click/type; always keep the producing pedal in view and tie chat-pane output to the pedal's look + scroll + carousel state) — folded into a Demo.cs refresh.

3. Hardware / GPU

AMD Ryzen 9 9900X (12 cores in WSL), 23 GiB RAM + 8 GiB swap, RTX 5070 12 GB GDDR7 Blackwell sm_120, driver 595.79, CUDA 13.2. The 12 GB GPU is now shared by three potential tenants (Ollama nemotron, vLLM twin-llama, Stable Diffusion); Reaper enforces single-resident swapping. Idle footprint ~0.9 GB; nemotron resident ~adds ~1 GB; vLLM 3B at util 0.70 ~10.6 GB.
