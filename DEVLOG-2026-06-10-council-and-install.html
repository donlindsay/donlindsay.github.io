DEVLOG SNAPSHOT — AI Stack — 2026-06-10 (council expansion + install capability)

This snapshot covers the cross-vendor council expansion into a registry-driven seat model and the SHELL software-install capability that was prepared and then exercised.

1. This session

  • Cross-vendor council, made registry-driven. Added an NVIDIA NIM seat nvidia-nano (Nemotron-3 nano-omni 30B) beside the 550B Ultra; operator policy is that the NVIDIA keys are promotional — used until they 401/quota-exhaust or terms change — and are metered at $0. Then generalized the two hardcoded seats into /opt/aistack/.nvidia-seats.json (mode 600, outside the repo): each {id, model, label, optional key} becomes a seat via council._make_nvidia_seat, so adding a hosted model is one JSON line. First per-seat-key entry is Meta Llama 3.3 70B.
  • Catalog fitness survey. Probed all 120 NVIDIA NIM catalog models for actual reachability across the three keys: 48 healthy, 58 not-provisioned (404 "not found for account" — NIM access is account-scoped), 11 degraded/hanging. Ranked healthy chat models by a transparent fitness score (size + capability + speed) and seated the two best new general voices, Mistral Large 3 675B and OpenAI gpt-oss 120B. Held back phi-4 (NVIDIA's phi-4 functions are degraded provider-side); operator excluded Qwen and DeepSeek. Council is now 8 reasoning seats across 5 vendors (Anthropic x3, Nemotron x2, Meta, Mistral, OpenAI).
  • Latent bug fixed. core.nvidia_chat hardcoded chat_template_kwargs {enable_thinking:false}, which Mistral tokenizers reject with HTTP 400; the toggle is now gated to nemotron/qwen ids only, making the registry genuinely model-agnostic.
  • SHELL software-install capability — prepared, then exercised. SHELL runs as the aishell user (HOME=/opt/aistack/work, login bash -lc). A new ~/.profile puts ~/.local/bin on PATH and sets the pipx env; install verbs (pipx/uv/gem/go) were added to the gate's _MUTATING so every install is confirm-gated while sudo apt stays denied unless the SUDO window is open; an INSTALLING SOFTWARE operating note codifies the policy (pipx for apps, venv/uv for libs, apt-in-SUDO for system; system python is PEP668 — never --break-system-packages); registered shell:SkillInstall. Proven with pipx install uv, then npm+npx 9.2.0 installed via the SUDO window and all 196 nvidia/skills (the npx skills CLI) into the aishell environment. Installing the same skills into the live Claude Code desktop app was blocked by the auto-mode safety classifier (self-modification + untrusted-code integration) even on operator request — that target is operator-run. A pre-existing broken dpkg package (nvidia-ai-workbench) surfaced and makes apt exit non-zero.

2. Lessons / TODO

  • NVIDIA NIM model access is account-scoped: a catalog id can 404 "not found for account", and the promotional keys are separate accounts.
  • Registry-driven seats were the right abstraction once the third model arrived — one JSON line each.
  • A hardcoded provider param (enable_thinking) broke a new vendor; gate provider-specific params by model family.
  • The safety classifier correctly refused to self-modify the live agent with untrusted skills.
  • TODO: adapt the SKILL.md skills into a SHELL-consumable form (the Nemotron agent does not read them); fix the broken nvidia-ai-workbench package.
