DEVLOG SNAPSHOT — AI Stack — 2026-06-10 (cost engineering)

This snapshot covers the provenance ontology and the neuro-symbolic cost-reconciliation of the SHELL cloud engine (the grounding and test-battery work have their own devlogs).

1. This session

  • SHELL.ttl — successor ontology to opus.ttl. A skills registry (shell:Skill as prov:Plan), a VoID map of the 18 live AllegroGraph repos, and three PROV agent identities (Operator / Claude / SHELL) bound to their git author + custody-actor slug. Distinguished the installed Claude desktop app from Claude-invoked-from-SHELL (owl:differentFrom — same model weights, different actor: one operates the machine, the other answers one council/cloud-engine prompt).
  • Model-identity correction. Anthropic swapped the desktop app from Fable 5 to Opus 4.8 server-side mid-session; the app cannot self-report its own model, so it had been mis-stamping commits. Corrected: git author now tracks the operator-confirmed model (Opus 4.8); today's 14 "Claude Fable 5" commits across the app + main repos were history-rewritten to "Claude Opus 4.8".
  • Council + spend meter. Added Opus 4.8 and Fable 5 as dedicated council peer seats. Built spend.py: one tel:CloudSpend event per cloud call with a caller tag (shell-engine / council-seat), measured token cost. This surfaced the finding that drove the rest of the day: the SHELL cloud engine sent ~2353 tokens of fixed apparatus (system prompt + tool defs) every turn, while a council seat sent 26 — a ~10x overhead that was not work.
  • Guards (deterministic, in the agent loop). A verbatim-URL guard (the local 4b re-types and corrupts URLs — youtube.ro); the confirm-gate Approve/Deny chips were rendering off-pane (fixed in the Avalonia app); failed-command stderr is now surfaced in the transcript and a typed approve/deny resolves a gate; the custody actor resolver now runs in all hooks, not just grounding.
  • Cost-reconciliation plan — fitness report + five phases, all shipped. P0 prompt caching (~80% off warm turns). P1 apparatus-as-graph substrate (apparatus.py assembles a per-class note + tool subset from the registry). P2 Nemotron triage trimming, ON by default after a battery regression gate cleared it (69% fewer apparatus tokens on cold turns, with an unknown-class fall-through to the full apparatus). P3 semantic memoization, ON (a repeat qa returns a cached answer from a vector store at zero cloud cost). P4 per-caller budget + per-class spend observability.

2. Lessons / TODO

  • The model a managed app runs on is externally controlled and not self-observable — the operator is the authority; do not bake the model into immutable commit authors.
  • P0 caching and P2 trimming are complementary by workload: caching wins warm multi-turn loops, trimming wins cold/single turns (a trimmed prefix is below the cache minimum).
  • TODO: precedent-widen apparatus.tools_for from observed per-class tool usage (only ever adds tools); finish ActNoteFromRegistry so the live operating note is graph-sourced.
