AI Stack — Complete Audit (2026-06-08)

A private, offline-first knowledge-and-tools workbench running natively in WSL2 Ubuntu.

1. Architecture — reasoning-core / language-shell on silicon

  • Symbolic brain (CPU / RAM): AllegroGraph 9.0.1 (free, 5M-triple license) + Allegro CL — RDF, SPARQL, RDFS++ reasoning, Prolog custody lineage.
  • Neural shell (GPU / VRAM, RTX 5070 12 GB): two serving engines in the active matrix — Ollama (nemotron-3-nano:4b + nomic-embed-text) and llama.cpp (llama-server, OpenAI API on :8002, built CUDA sm_120 Blackwell-FP4; serves the Ollama GGUF blobs — PRIMARY, default llama.cpp × Nemotron) — selectable as a free engine × model matrix; plus a socket-activated diffusers Stable Diffusion worker. vLLM is SHELVED (dormant, swap-in-able). Reaper governs the shared 12 GB (single-resident swapping).
  • Edge glue: Python package aistack (shrinking).
  • n8n: EVICTED from the runtime; preserved as an optional swap-in library.

3a. Local LLM — engine × model matrix + Reaper console

  • Two axes (pedalboard slots 0 + 1): Engine ∈ {Ollama, llama.cpp} × Model ∈ {Nemotron, Twin Llama}, selected independently (free matrix); default = llama.cpp × Nemotron. localllm._resolve() maps each pair to a backend with graceful Ollama fallback; _MANAGED_ENGINES enforces single-resident GPU (engaging one evicts Ollama + stops the other). POST /local-engine {engine?,model?}. (vLLM is shelved but still resolvable via set_engine("vllm").)
  • llama.cpp on Blackwell sm_120: llama-server built from source (-DGGML_CUDA=ON -DCMAKE_CUDA_ARCHITECTURES=120; ARCHS=1200, Blackwell-native FP4). Serves the exact Ollama GGUF blobs from /opt/aistack/models/gguf/ (parity), ~180 tok/s; ai-llamacpp systemd unit on :8002, full GPU offload, model-swap via .target-model. Serving infra in deploy/llamacpp/. (Shelved) vLLM: 0.22.1 + torch 2.11.0+cu130, deploy/vllm/, dormant.
  • Reaper (slot 4) = VRAM/perf console: live GPU telemetry (GET /gpu via nvidia-smi — VRAM/util/temp/power), the 3×2 status matrix (GET /local-matrix: live/loadable/fallback/placeholder), Load/Release vLLM, and per-cell benchmarks (POST /local-bench) — Ollama path uses native ns timing (e.g. nemotron ≈ 133 tok/s, TTFT 26 ms), vLLM path wall-clock + completion tokens.

4. Cloud web-tier council — 8 peers

Fan-out in parallel; synthesized locally by nemotron (only the query leaves the box).

  • Search: serpapi (direct) · ddgs (keyless) · perplexity (direct)
  • Reasoning: claude (Anthropic) · openai (gpt-4o-mini) · openrouter (gemini-2.5-flash) · huggingface (llama-3.3-70b) · ollama (gpt-oss:120b, via the linked box)
  • Reachable via the web_council tool, POST /council, and the WEB→CLOUD knob.

5. Key store (7 secrets + device keypair, all chmod 600)

.anthropic .openai .openrouter .perplexity .serpapi .huggingface .ollama + /root/.ollama/id_ed25519 (registered on ollama.com). All council/search paths now call vendors direct. Full dependency graph materialized in registry (<http://kg.local/inv/graph>), custody-linked.

6. Models

  • Local — Ollama: nemotron-3-nano:4b (agent + council synthesis), nomic-embed-text (embeddings).
  • Local — llama.cpp: nemotron + twin-llama from the Ollama GGUF blobs (GPU, :8002, primary). (Shelved) vLLM: twin-llama = unsloth/Llama-3.2-3B-Instruct (dormant).
  • Cloud: gpt-4o-mini, claude-haiku-4-5, gemini-2.5-flash, llama-3.3-70b, gpt-oss:120b, perplexity-sonar; SerpAPI + DuckDuckGo search.

7. n8n — purged

Services stopped + removed from start-stack.sh; n8n-data, the Python runner, and the nvm Node 22 it ran on are all deleted (~3.2 GB freed), and its encrypted credential store destroyed. Code survives as aistack/n8n.py and the Search pedal keeps an SerpAPI ↔ n8n swap, but the n8n variant is now a dead skin (no Node/workflows) — like the ComfyUI image skin. (System node v18 stays: it is a pandoc dependency, unrelated to n8n.)

8. Pedalboard (Avalonia frontend) — 6-slot board

0 top → 5 bottom, two slots can't be empty: 0 Engine (Ollama/Llama.cpp), 1 Model (Nemotron/Twin Llama), 2 Knowledge (carousel faces Ask/Search/Graph/KG), 3 Media (SD/ComfyUI), 4 Reaper, 5 Audio/VXA. Pedal-slot is decoupled from content-tab; double-click swaps a slot's carousel variant. (Demo --demo screencast refresh for the 6-slot board + natural-capture choreography is in flight as a separate task.)
