AI Stack — Architecture

id: urn:kg:doc:architecture · source: guide · date: 2026-06-08

A concise picture of how the pieces fit. (For the dated, detailed snapshot see the devlogs.)

Engine

The local model runs on a small engine × model matrix — the sole live engine is llama.cpp (llama-server, OpenAI-compatible on :8002, built for CUDA sm_120 / Blackwell). The default quadrant is llama.cpp × Nemotron (nemotron-3-nano:4b), with Twin-Llama as a second model. The 12 GB GPU is single-resident. Ollama, vLLM, n8n and ComfyUI are shelved add-ins — out of the active matrix, code preserved and swap-in-able.

Knowledge graph + retrieval

AllegroGraph 9.0.1 holds documents (data.docs), agent memory (mem.agent), provenance (prov.custody), durable SHELL sessions (shell.sessions), and more, organized with SKOS / Dublin Core / PROV / DCAT. Its native vector stores (data.docs-vec, mem.agent-vec) embed via nomic-embed-text — served by an in-process llama.cpp embedding adapter that speaks the Ollama embeddings API on :11434, where AllegroGraph's embedder points. So one runtime — llama.cpp — covers both chat and embeddings, with no Ollama binary, and the vectors stay in the graph.

Service layer

A FastAPI backend on :8765 fronts the agent, the skills, and the SHELL pedal. Every front end (pedalboard, desktop, web, CLI) is a thin client over it. WSL forwards localhost to Windows.

SHELL + self-building

The SHELL pedal is an agentic chat-editor: one engine-agnostic driver runs a tool loop over run_command plus typed file tools (read_file / write_file / apply_patch) under a permission policy — read-only auto, mutating confirm (a human-approved gate), destructive/secret deny. It runs on the local model or, opt-in, on Opus 4.8. Sessions persist to AllegroGraph and survive restarts. Pointed at its own source via /shell/selfbuild, the agent can propose or apply changes to the stack itself — minimal edit, compile-check, then stage for a human-approved commit, never pushing. Every operation is written to the custody graph (PROV) — explainable automation, and explainable self-modification.
