AI Stack — Roadmap

State as of the great consolidation (2026-06-04). Everything below the line is secured; everything above is the next evolution.

✅ Secured baseline (the flanks)

  • Native, no Docker — AllegroGraph, n8n + Python runner, Ollama, ComfyUI, the

unified main/ app (desktop/web/cli) all run as native WSL2 processes.

  • One model, one path — nemotron-3-nano:4b (2.8 GB, NVIDIA/Blackwell) does

everything: the agent (all 11 tool-calls, routerless — the system prompt handles restraint), generation/extraction, and AllegroGraph's askMyDocuments (documents) + askForTable (tables). nomic-embed-text for vectors. No router, no model selector.

  • Sharpened system prompts — shared IDENTITY voice + restraint + synthesis guidance

(Track B, local half, done).

  • Cloud batphone — ask_claude / perplexity_search via encrypted n8n credentials,

opt-in, off-box, flagged. Keys only in chmod-600 files, never in git.

  • One app, three faces — aistack package + desktop (Qt, frameless skin) / web

(Gradio) / cli (truecolor). UI-scale slider, Windows tray + shortcuts.

  • Verified green end-to-end; ~31 GB reclaimed; repos clean; docs current.

Next evolution — candidate tracks

A. Face outward: MCP server (strategic; ~an afternoon)

Wrap the 11 skills (AG RAG, graph_query, extract, web_search, image, …) as a Model Context Protocol server. Inverts the relationship — Claude Desktop / Claude Code / any MCP client can use the local AllegroGraph + knowledge graph as tools. The stack stops being only a consumer and becomes a capability provider. Also lets the app consume other MCP servers as new skills for free.

B. Sharpen the brain (the "bootstrap brain")

  • System prompts — ✅ done: a shared IDENTITY voice + tool restraint + synthesis

guidance, which also let the router be retired (the prompt handles restraint).

  • 3rd escalation tier (remaining; optional, cloud) — promote ask_claude into a

hard → Claude escalation for multi-step chains / synthesis where the 4 B is thin. Model-tiered (Haiku→Sonnet→Opus), prompt-cached. Deferred as cloud-leaning.

C. Deepen the knowledge: ingestion & corpus quality

askMyDocuments is only as good as the docs — the current corpus is browse-history, so answers run shallow. Richer ingestion via the Sponger rack, more doc types (PDF/structured), re-embedding, and dedup. Makes the RAG actually authoritative.

D. Harden for keeps: resilience & ops

  • Backups — scheduled AG repo exports (.nt) + n8n workflow export.
  • Auto-start — optional Startup-folder entry for the tray / ai-desktop unit.
  • Fallback on file — qwen3:8b (Apache + thinking) is the re-pullable step-up if

nemotron's 4 B ever proves thin on hard multi-step turns.

  • Monitoring — surface the status line + a tiny health webhook.

E. Identity & polish

The guitar-pedal window skin (the frameless title bar + skin pipeline already exist); later, a native-Windows toolkit if WSLg/xcb ever chafes.

Suggested order

B's local half is done. Next: C (corpus depth) or D (resilience) as the stack earns real use; A (MCP) when you want reach (mind the local→cloud boundary it opens); B's cloud escalation tier and E (the guitar-pedal skin) whenever the mood strikes.
