The Local AI Stack — Guide

A fully self-hosted knowledge-graph + RAG + automation + image-generation platform, running natively in WSL2 Ubuntu on a single workstation. No cloud, no Docker in the data path, no data leaving the machine.

1. Introduction

This stack turns one GPU workstation into a private AI platform. It combines a knowledge graph (AllegroGraph), local LLMs (Ollama), a workflow engine (n8n), image generation (ComfyUI), and a web-extraction pipeline into a set of simple HTTP endpoints you can call from scripts, the browser, or other tools.

Design principles

  • Local-first & private — every model, every byte of data, and every inference

runs on the box. Nothing is sent to a third party. Instance identifiers use urn: IRIs, services bind to localhost, and the one root-shell capability that existed was deliberately removed.

  • Native, not containerized — everything runs as host processes (systemd units +

daemons). Adding a Python library to a workflow is one pip install, not an image rebuild.

  • Composable — each capability is a webhook. Chain them in n8n, call them from

curl, or wire them into your own apps.

  • Standards-based — the graph uses W3C vocabularies (RDF, SPARQL, SKOS, Dublin

Core, SHACL); the LLM endpoints speak plain JSON.

  • Version-controlled — the code lives in a git repo with ruff/mypy/bandit

pre-commit hooks.

Hardware: RTX 5070 (12 GB, Blackwell) · 23 GB RAM · WSL2 Ubuntu.

Bring it all up with one command:


bash /opt/aistack/start-stack.sh

Or use the Console — one app with three faces over a shared backend: a native desktop window (Qt), the web UI at http://localhost:7860, and a terminal CLI (bash /opt/aistack/main/run.sh [desktop|web|cli]). Each has a Chat agent (nemotron calls the capabilities below as tools, with persistent AllegroGraph-backed memory across sessions) plus Ask·RAG, Search, Extract, Image, and Graph tabs, and a live service-status bar.

2. Feature set

Knowledge graph (AllegroGraph)

  • RDF quad-store with full SPARQL 1.1, reachable over HTTP.
  • Native vector store — llm:nearestNeighbor and llm:askMyDocuments magic

predicates do retrieval + generation + citation inside a single SPARQL query.

  • Gruff visual graph explorer at /gruff.
  • SHACL validation (shapes.ttl) to keep the graph clean.
  • Standards-based modeling: Dublin Core + SKOS + RDF, urn: instance IRIs,

embeddings stored as rdf:JSON literals in a hidden graph (so Gruff stays readable but retrieval still works).

  • Repositories: browser_rag (from browser history), graphrag, n8n_rag

(the working RAG store), n8n_rag-vec (native vector store).

GraphRAG — two engines

  • Portable RAG (/webhook/agraph-rag) — Ollama embeddings + stdlib cosine in

Python. Modes: store, query, ask (grounded answer), reindex.

  • Native RAG (/webhook/agraph-rag-native) — AllegroGraph's own vector store +

askMyDocuments for retrieve-generate-cite in one query, with source citations.

Local LLMs (Ollama)

  • nemotron-3-nano:4b — the single model for everything: all 11 agent tool-calls

(routerless — the agent's system prompt handles tool restraint), generation / entity-extraction, AND AllegroGraph's askMyDocuments (documents) + askForTable (tables). NVIDIA, 2.8 GB, runs on the Blackwell GPU — the only local model that passes Franz's function-call parser for the table predicate.

  • nomic-embed-text — 768-dim embeddings for all RAG.

Workflow automation (n8n)

  • Webhooks and hosted chat UIs, all built by idempotent n8n/build_*_wf.py

scripts in the repo.

  • A native Python task-runner: workflow Code nodes run in a host venv, so they

can use any installed Python package (no container, no image rebuild).

Web extraction — Sponger-style RDFizer (extract/)

A native reimplementation of Virtuoso's "cartridge" pattern. Turns a list of URLs into RDF in the graph via three best-effort tiers:

  1. extruct — JSON-LD / microdata / RDFa / OpenGraph structured data,
  2. trafilatura — main content + metadata,
  3. nemotron — salient entities.

Each page becomes a retrievable RAG document plus knowledge-graph metadata and entity nodes. Bespoke per-domain extractors plug into a registry.

Web search

  • ddgs (DuckDuckGo) as an open webhook + a hosted chat UI.

Image generation (ComfyUI)

  • SD1.5 text-to-image via a webhook; returns a viewable image URL + host path.

Operations

  • One-command startup; systemd-managed services survive logout.
  • Git repo + pre-commit (ruff/mypy/bandit). CUDA 12.8 toolkit available for GPU

source builds.

3. Examples

All endpoints are POST … application/json on localhost.

Start / stop / inspect


bash /opt/aistack/start-stack.sh                      # bring everything up
systemctl stop ai-n8n ai-n8n-runner                   # stop n8n + runner
journalctl -u ai-n8n -f                               # tail n8n logs

Knowledge graph — SPARQL & size


# How many triples in a repo?
curl -s -u admin:admin http://localhost:10035/repositories/n8n_rag/size

# Ad-hoc SPARQL
curl -s -u admin:admin http://localhost:10035/repositories/n8n_rag \
  --data-urlencode 'query=SELECT ?s ?o WHERE { ?s <http://purl.org/dc/terms/title> ?o } LIMIT 5' \
  -H 'Accept: text/csv'

Generate text (Ollama)


curl -s -X POST http://localhost:5678/webhook/ollama-gen \
  -H 'content-type: application/json' \
  -d '{"prompt":"Explain knowledge graphs in two sentences.","model":"nemotron-3-nano:4b"}'

RAG — portable engine


# Semantic search over a repo
curl -s -X POST http://localhost:5678/webhook/agraph-rag \
  -H 'content-type: application/json' \
  -d '{"mode":"query","text":"vector database","repo":"n8n_rag","k":3}'

# Grounded answer (retrieve + generate)
curl -s -X POST http://localhost:5678/webhook/agraph-rag \
  -H 'content-type: application/json' \
  -d '{"mode":"ask","text":"What is vLLM?","repo":"n8n_rag","k":4}'

# Store a new document
curl -s -X POST http://localhost:5678/webhook/agraph-rag \
  -H 'content-type: application/json' \
  -d '{"mode":"store","text":"AllegroGraph is an RDF triplestore.","repo":"n8n_rag"}'

RAG — native engine (citations)


curl -s -X POST http://localhost:5678/webhook/agraph-rag-native \
  -H 'content-type: application/json' \
  -d '{"mode":"ask","text":"What topics are in the documents?","k":3}'
# -> {"engine":"...askMyDocuments (nemotron-3-nano:4b)","answer":"...","citations":[...]}

Web search


curl -s -X POST http://localhost:5678/webhook/websearch \
  -H 'content-type: application/json' \
  -d '{"query":"allegrograph rdf database","k":3}'

Extract websites into the graph (Sponger rack)


curl -s -X POST http://localhost:5678/webhook/extract \
  -H 'content-type: application/json' \
  -d '{"urls":["https://docs.vllm.ai","https://en.wikipedia.org/wiki/RDF"],
       "repo":"n8n_rag","limit":10,"llm":true}'
# Each URL -> structured data + content + entities, stored and immediately
# searchable via /webhook/agraph-rag.

Generate an image (ComfyUI)


curl -s -X POST http://localhost:5678/webhook/comfy-image \
  -H 'content-type: application/json' \
  -d '{"prompt":"a serene mountain lake at sunset, highly detailed","steps":20}'
# -> {"view_url":"http://localhost:8188/view?...","host_path":"/opt/aistack/ComfyUI/output/..."}

Chat UIs (in the browser)

Open any of these for a hosted chat interface:


http://localhost:5678/webhook/chat-ollama/chat     # chat with nemotron-3-nano:4b
http://localhost:5678/webhook/chat-rag/chat        # grounded GraphRAG chat (citations)
http://localhost:5678/webhook/chat-image/chat      # describe -> image
http://localhost:5678/webhook/chat-search/chat     # web-search chat

Programmatic message: POST /webhook/<path>/chat with {"action":"sendMessage","sessionId":"abc","chatInput":"hello"}.

Add a Python library to workflows


/opt/aistack/n8n-runner/venv/bin/pip install <package>
systemctl restart ai-n8n-runner
# the package is now importable in any n8n Python Code node

4. Where things live

  • agraph/ — AllegroGraph install + data · n8n-data/ — n8n DB + credentials
  • n8n-runner/ — native task runner (venv + launcher wrapper)
  • graphrag/ — git repo: pipeline, n8n/build_*_wf.py builders, extract/ rack, shapes.ttl
  • ComfyUI/, models/ — image gen + model caches
  • start-stack.sh · README.md (manifest) · this GUIDE.md

Secrets (.n8n-api-key, .n8n-runner-token, the encryption key) stay on disk, outside git. See README.md for the full layout and service-management details.
