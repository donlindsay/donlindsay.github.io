Notes

  • vLLM×Nemotron is now nvidia/Llama-3.1-Nemotron-Nano-4B-v1.1 (pure Transformer, ~56 tok/s; util 0.90 / max-len 5120, FLASH_ATTN). It's reasoning-first, so localllm.generate/bench prepend NVIDIA's "detailed thinking off" (C.VLLM_THINKING_OFF) + strip <think> → fast, clean answers. Tool-calling enabled (nemotron in C.VLLM_TOOL_MODELS) via NVIDIA's llama_nemotron_json parser plugin + tool chat template, wired in serve.sh — Phase C confirms it drives the full agent loop. _(History: it replaced the NemotronH Mamba-hybrid BF16, ~10 tok/s, whose FP8 build was blocked by flashinfer's missing sm_120 GEMM.)_
  • Ollama×Twin and ×Nemotron share the Ollama daemon (fast). Phase B routes the raw LLM step via localllm.generate (engines genuinely differ); Phase C drives the production run_agent so tool ROUTING differs per quadrant.
