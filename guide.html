AI Stack — Guide

id: urn:kg:doc:guide · source: guide · date: 2026-06-08

How to run the stack, what the front ends are, and how to drive the pedals.

Run it

One script brings the whole stack up inside WSL2 Ubuntu:


bash /opt/aistack/start-stack.sh

That starts AllegroGraph (:10035), llama-server (:8002), the embedding adapter (:11434, in place of the old Ollama daemon), and the FastAPI backend (:8765). WSL forwards localhost, so Windows clients reach the API with no extra setup.

Front ends

Everything is a thin client over the http://127.0.0.1:8765 API:

  • Pedalboard — the native Avalonia desktop app (the primary UI).
  • Desktop — a PySide6/Qt frame; Web — a Gradio tab set; CLI — a truecolor REPL

(/opt/aistack/main/run.sh cli).

The pedals

Each pedal has a mode knob (MAIN / OPTS / SYSTEM):

  • Chat / Ask — converse with the local model, with or without memory.
  • Search / Graph / KG — RAG retrieval, SPARQL, and knowledge-graph views over AllegroGraph.
  • Image — Stable Diffusion generation.
  • VXA — voice and audio.
  • SHELL — an agentic chat-editor: ask in plain language and the agent runs shell commands and

edits files.

SHELL quickstart

The ACCESS ▸ SHELL knob picks the environment (Ubuntu bash or Windows PowerShell). Read-only work runs automatically; anything that mutates files or state pauses for your approval (an Approve / Deny chip); destructive or secret-touching actions are denied outright. Sessions are durable — the transcript and any pending approval survive a restart. Set ACCESS ▸ WEB = CLOUD to drive SHELL with Opus 4.8 instead of the local model.
